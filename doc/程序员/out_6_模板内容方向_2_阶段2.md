让视频有"节奏起伏"：

### Step 3：实现节奏规则引擎
**做什么**：
- 扩展 `TimelineCalculateService`
- 根据 `scene_type` 应用不同的节奏规则
- 实现 3 种规则的具体逻辑

**产出**：
```python
# backend/app/services/timeline_calculate_service.py
def calculate_timeline_with_rhythm(scene: Scene, tts_metadata: dict) -> dict:
    """根据场景类型应用节奏规则"""
    # 获取节奏规则
    rule = RHYTHM_RULES.get(scene.scene_type, RHYTHM_RULES["explanation"])

    # 基础时间轴计算
    timeline = calculate_timeline(scene, tts_metadata)

    # 应用节奏规则
    timeline = apply_rhythm_rule(timeline, rule, scene.duration_sec)

    return timeline

def apply_rhythm_rule(timeline: list, rule: dict, duration: float) -> list:
    """应用节奏规则调整时间轴"""
    if rule["emphasis_density"] == "high":
        # Hook：关键词密集，调整间隔
        timeline = adjust_intervals(timeline, min_interval=rule["min_interval"])

    elif rule["emphasis_density"] == "paired":
        # 对比：关键词成对出现
        timeline = pair_keyframes(timeline)

    # 调整动效速度
    for kf in timeline:
        if rule["effect_speed"] == "fast":
            kf["duration"] = 0.2
        elif rule["effect_speed"] == "medium":
            kf["duration"] = 0.3

    # 添加结尾停顿
    if rule["pause_after"]:
        timeline.append({
            "time": duration - 0.5,
            "element": "",
            "action": "pause",
            "duration": 0.5
        })

    return timeline
```

### Step 5：集成主流程（使用节奏规则）
**做什么**：
- 修改视频生成主流程
- 调用 `calculate_timeline_with_rhythm` 而不是 `calculate_timeline`

**产出**：
```python
# backend/app/services/video_generate_service.py
async def generate_video(article: str, hook: str):
    scenes = await scene_generate_service.generate(article, hook)

    for scene in scenes:
        tts_result = await tts_service.generate_audio(scene.voiceover)
        scene.tts_metadata = tts_result

        # 使用节奏规则计算时间轴（新）
        timeline = timeline_calculate_service.calculate_timeline_with_rhythm(
            scene, tts_result
        )
        scene.timeline_data = timeline

        await scene.save()

    return await render_service.render(scenes)
```

## 2. 节奏规则详细设计

### Hook 场景规则
```python
{
    "emphasis_density": "high",      # 关键词密集
    "effect_speed": "fast",          # 0.2 秒
    "pause_after": True,             # 结尾停顿
    "min_interval": 0.3              # 最小间隔
}
```

**效果**：
- 关键词快速出现，制造紧迫感
- 动效快速，吸引注意力
- 结尾停顿，给观众反应时间

### Explanation 场景规则
```python
{
    "emphasis_density": "medium",    # 关键词均匀
    "effect_speed": "medium",        # 0.3 秒
    "pause_after": False,
    "min_interval": 0.8              # 最小间隔
}
```

**效果**：
- 关键词均匀分布，节奏平稳
- 动效适中，不抢戏
- 无停顿，流畅过渡

### Contrast 场景规则
```python
{
    "emphasis_density": "paired",    # 关键词成对
    "effect_speed": "alternating",   # 交替
    "pause_after": False,
    "min_interval": 0.5              # 最小间隔
}
```

**效果**：
- 关键词成对出现，制造对比
- 动效交替（第一个快，第二个慢），强化对比
- 无停顿，保持连贯

## 4. 关键风险与应对

### 风险 1：LLM 场景类型标注不准确
**影响**：规则应用错误，节奏不对
**应对**：
- 在 Prompt 中增加更多示例
- 如果准确率 < 80%，改为规则判断（关键词匹配）

### 风险 2：规则参数不合适
**影响**：节奏起伏不明显
**应对**：
- 人工测试 20 个视频
- 根据反馈调整参数
- 记录最佳参数组合

### 风险 3：用户感知不明显
**影响**：用户认为节奏没有起伏
**应对**：
- 加大参数差异（fast=0.15, medium=0.3）
- 增加视觉提示（不同颜色）

---

## 5. 验收标准（阶段 2）

### 技术指标
- LLM 场景类型标注准确率 ≥ 80%
- 规则应用成功率 = 100%（纯计算）

### 业务指标
- 用户认为"节奏有起伏"的比例 ≥ 50%
- 用户修改率下降 ≥ 10%（从 30% 降到 20%）

### 如果失败
- 如果标注准确率 < 80%，改为规则判断
- 如果用户感知 < 50%，加大参数差异
- 如果修改率没下降，停止开发
